## Proposal Report

### 動機與目標

#### 動機
大概是昨天我發現我又忘記寫webwork微積分作業了，雖然只有很低的分數，但我心還是很痛，所以我立志要弄出一個專門提醒我deadline要到的系統，其他的日曆app都被我拿來當大事件的紀錄而不是作業截止時間，我覺得混在一起好亂，所以我想弄一個自己的系統（ntu cool好多雜七雜八的通知）。

#### 目標
1. 建立一個可新增、刪除、查詢與修改任務的截止日管理系統。
2. 使用紅黑樹作為核心資料結構，維持任務的有序排列。
3. 提供依照 deadline 顯示所有任務與查詢最近到期任務的功能。
4. 說明此系統與資料結構、進階程式設計及 Linux kernel 設計概念之間的關聯。


### 預期功能
1. **新增任務**
   - 使用者可輸入任務名稱、課程名稱、deadline、優先級與備註。
   - 系統會將任務插入紅黑樹中，依 deadline 維持排序。

2. **刪除任務**
   - 使用者可刪除已完成或不需要的任務。
   - 系統需正確更新紅黑樹結構。

3. **修改任務**
   - 可修改任務內容，尤其是 deadline。
   - 若 deadline 改變，需重新維持資料的有序性。

4. **查詢最近到期任務**
   - 可快速顯示目前最接近截止時間的任務。

5. **列出所有任務**
   - 依 deadline 由近到遠顯示所有任務。

6. **標記完成狀態**
   - 可將任務標記為已完成或未完成。

7. **進階功能（若時間允許）**
   - 任務分類（作業、考試、報告、生活提醒）
   - 搜尋特定課程或關鍵字
   - 匯出任務資料到文字檔案
   - 比較紅黑樹與其他資料結構在此情境下的效率差異

### 與課程的關聯
- **資料結構方面：**
  - 本專題的核心資料結構是紅黑樹（Red-Black Tree）。
  - 紅黑樹屬於 self-balancing binary search tree，可在搜尋、插入與刪除操作中維持 $O(\log n)$ 的時間複雜度。
  - 由於本專題需要管理大量依 deadline 排序的任務，且任務會持續新增、刪除與修改，因此紅黑樹非常適合這種「有序動態集合」的問題。

- **演算法與結構觀念方面：**
  - Binary Search Tree 的節點關係與搜尋方式
  - 紅黑樹的平衡規則
  - 旋轉（rotation）與重新著色（recoloring）
  - 依序走訪（in-order traversal）以輸出排序結果
  - 最小值查詢，以取得最近到期任務

- **進階程式設計方面：**
  - 本專題需要自行設計資料結構、類別與函式分工，例如任務類別、紅黑樹類別與使用者操作介面。
  - 這能練習模組化設計、抽象化思考、程式可讀性與維護性。

- **與實際系統設計的關聯：**
  - 本專題也參考了 Linux kernel 在管理時間排序事件時所使用的設計概念。
  - Linux kernel 在某些場景中會使用紅黑樹管理需依時間排序的事件。
  - 本專題則將這種系統層級的概念轉化為學生日常生活中的截止日管理問題，讓資料結構的應用更具體且更生活化。
